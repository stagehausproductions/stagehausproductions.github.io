## Project Overview

StageHaus Productions website — a fully static site generated by [Eleventy 2.x](https://www.11ty.dev/) using Nunjucks templating. All pages share a common base layout, header, footer, and design system. The site is optimized for GitHub Pages deployment via GitHub Actions.

**Tech Stack:**
- **Generator:** Eleventy 2.0.1
- **Templating:** Nunjucks (.njk)
- **CSS:** Vanilla CSS with custom properties design system
- **JS:** Vanilla ES6+ (no frameworks, no bundler)
- **Deployment:** GitHub Pages via GitHub Actions
- **Fonts:** Google Fonts (Cormorant Garamond + Inter + JetBrains Mono)
- **Payments (scaffold):** Square Web Payments SDK / Stripe.js

---

## Prerequisites

- **Node.js** 18 or higher — [nodejs.org](https://nodejs.org/)
- **npm** 9 or higher (comes with Node.js)
- A **GitHub account** with a repository for the site

## Local Development

```bash
# 1. Clone the repository
git clone https://github.com/YOUR_USERNAME/stagehaus.git
cd stagehaus

# 2. Install dependencies
npm install

# 3. Start the development server
npm start
# → Runs at http://localhost:8080 with live reload
```

**Available scripts:**

| Command | Description |
|---|---|
| `npm start` | Start dev server at localhost:8080 with watch |
| `npm run dev` | Alias for `npm start` |
| `npm run build` | Production build → `_site/` |
| `npm run clean` | Delete `_site/` directory |

The dev server watches all source files and reloads the browser automatically.

## Deploying to GitHub Pages

### First-Time Setup

1. **Create a GitHub repository** for the site (public or private)

2. **Enable GitHub Pages** in the repo settings:
   - Go to `Settings → Pages`
   - Under **Source**, select **GitHub Actions**

3. **Push the code:**
   ```bash
   git init
   git remote add origin https://github.com/YOUR_USERNAME/stagehaus.git
   git add .
   git commit -m "Initial commit — StageHaus site"
   git push -u origin main
   ```

4. The GitHub Actions workflow (`.github/workflows/deploy.yml`) automatically:
   - Installs Node dependencies
   - Runs `npm run build`
   - Deploys the `_site/` output to GitHub Pages

5. **Your site will be live at:** `https://YOUR_USERNAME.github.io/stagehaus/`

### Custom Domain

1. Add your domain in `Settings → Pages → Custom domain`
2. Create a `CNAME` file in `src/` containing your domain:
   ```
   stagehaus.com
   ```
3. Add the file to Eleventy's passthrough in `.eleventy.js`:
   ```js
   eleventyConfig.addPassthroughCopy("src/CNAME");
   ```
4. Configure your DNS: add a CNAME record pointing to `YOUR_USERNAME.github.io`

### Pathprefix (Subdirectory Deployments)

If the site is deployed at a subdirectory (e.g., `github.io/stagehaus/`), the base URL for assets must match. Check `.eleventy.js` for the `pathPrefix` setting and update if needed.

## Adding New Pages

### Phase 1 Page (Full Content)

1. Create `src/your-page-name.njk`
2. Use this front matter template:
   ```yaml
   ---
   layout: base.njk
   title: Page Title — StageHaus Productions
   pageDescription: SEO description (150–160 chars)
   currentPage: your-page-name
   bodyClass: page-your-page-name
   tags: phase1
   ---
   ```
3. Import macros at the top:
   ```njk
   {% from "macros/components.njk" import pageHero, processStep, valueCard %}
   ```
4. Use `{{ pageHero("Label", "Title", "Subtitle", "Breadcrumb", "/url/") }}` for the hero
5. The page is automatically added to the `phase1` collection and will build to `/your-page-name/index.html`

### Using the CTA Band Partial

Set these variables before including `cta-band.njk`:
```njk
{% set ctaSectionLabel = "Label text" %}
{% set ctaHeading = "Main heading" %}
{% set ctaSubtext = "Supporting paragraph" %}
{% set ctaPrimaryLabel = "Primary button text" %}
{% set ctaSecondaryLabel = "Secondary button text" %}
{% set ctaSecondaryUrl = "/target-url/" %}
{% include "partials/cta-band.njk" %}
```

## Activating Phase 2 Pages

Phase 2 pages are already built and deployed as "Coming Soon" placeholders. To convert a Phase 2 page to a full content page:

1. **Update the page file** (e.g., `src/event-production.njk`):
   - Replace the `coming-soon` section with full page content
   - Change `tags: phase2` to `tags: phase1`

2. **Update `src/_data/navigation.json`:**
   - Find the service entry (e.g., `"key": "event-production"`)
   - Set `"comingSoon": false`
   - Set `"phase": 1`

3. **Rebuild and deploy:**
   ```bash
   npm run build
   git add . && git commit -m "Activate Event Production page" && git push
   ```

The navigation mega-menu automatically moves the service from "Coming Soon" to "Active Services" based on the `comingSoon` flag.

## Artist Template Usage

The `src/artist-template/` directory contains a standalone static HTML/CSS artist site template ("Midnight" variant). It requires no build step.

### Deploying a Template for an Artist

1. **Copy the template directory** to a new folder:
   ```bash
   cp -r src/artist-template/ my-artist-site/
   ```

2. **Customize `index.html`** — replace all `[PLACEHOLDER]` values:
   - `[ARTIST NAME]` — artist or band name
   - `[ARTIST GENRE]` — genre and location string
   - `[ARTIST BIO]` — biography paragraphs
   - `[ALBUM TITLE]`, `[SINGLE TITLE]`, etc. — music catalog
   - Show dates in the `#shows` section
   - Social media links in the nav and footer

3. **Add real images:**
   - Create an `images/` folder inside the template directory
   - `images/hero-bg.jpg` — hero background (1920×1080px, compressed WebP preferred)
   - `images/artist-photo.jpg` — about section portrait
   - `images/album-art.jpg` — release artwork
   - Update `style.css` and `index.html` to reference these files

4. **Deploy options:**
   - **GitHub Pages:** Push to a repo, enable Pages from the root or `/docs` folder
   - **Netlify Drop:** Drag the folder to [netlify.com/drop](https://app.netlify.com/drop)
   - **Vercel:** `npx vercel` from the template directory

### Adding to the StageHaus Template Page

To display the template as a new card on the `/website-development/` page:
1. Add a new `.artist-template-card` block in `src/website-development.njk`
2. Update the template grid in `src/assets/css/style.css` if needed

## Merch Store Integration

The `/merch-operations/` page includes a scaffold for an artist storefront. To wire up real payments:

### Square Web Payments SDK

```html
<!-- Add to base.njk or page <head> -->
<script src="https://web.squarecdn.com/v1/square.js"></script>
```

```js
// In main.js or a page-specific script
const payments = Square.payments('YOUR_APP_ID', 'YOUR_LOCATION_ID');
// Attach payment form to .merch-card__buy buttons
```

Full docs: [developer.squareup.com/docs/web-payments/overview](https://developer.squareup.com/docs/web-payments/overview)

### Stripe

```html
<script src="https://js.stripe.com/v3/"></script>
```

```js
const stripe = Stripe('YOUR_PUBLISHABLE_KEY');
// Use Stripe Payment Element or Checkout
```

Full docs: [stripe.com/docs/payments/payment-element](https://stripe.com/docs/payments/payment-element)

### Product Data via JSON

To drive products from data instead of hardcoded HTML:

1. Create `src/_data/merch.json`:
   ```json
   {
     "products": [
       { "title": "Classic Logo Tee", "price": "$35.00", "sku": "tee-001", "image": "tee.jpg", "meta": "Unisex · XS–3XL", "inStock": true },
       { "title": "Tour Hoodie", "price": "$75.00", "sku": "hoodie-001", "image": "hoodie.jpg", "meta": "Heavyweight · S–2XL", "inStock": true }
     ]
   }
   ```

2. Loop in `merch-operations.njk`:
   ```njk
   {% for product in merch.products %}
     {% if product.inStock %}
     <div class="merch-card">
       <div class="merch-card__img"><img src="/assets/images/merch/{{ product.image }}" alt="{{ product.title }}" /></div>
       <div class="merch-card__body">
         <h3 class="merch-card__title">{{ product.title }}</h3>
         <p class="merch-card__meta">{{ product.meta }}</p>
         <div class="merch-card__footer">
           <span class="merch-card__price">{{ product.price }}</span>
           <button class="merch-card__buy btn btn--primary btn--sm" data-product="{{ product.sku }}">Add to Cart</button>
         </div>
       </div>
     </div>
     {% endif %}
   {% endfor %}
   ```

## Content Updates

### Update Site Info

Edit `src/_data/site.json` for global changes:
- Company name, tagline, contact info, social links
- Stats displayed in the hero and stat bars

### Update Navigation

Edit `src/_data/navigation.json`:
- Add, remove, or reorder nav items
- Update service taglines and icons
- Toggle `comingSoon` to activate/deactivate Phase 2 pages

### Add Team Members

Edit `src/about.njk` — find the `.grid-4` team section and add a new:
```njk
{{ teamCard("XX", "Name", "Role", "Bio text here.") }}
```

### Update Testimonials

Each testimonial uses the `testimonialCard` macro:
```njk
{{ testimonialCard("Quote text", "Name", "Role / Band", "Initials") }}
```

## Environment & Configuration

### `.eleventy.js` Key Settings

| Setting | Value | Description |
|---|---|---|
| Input dir | `src` | All source files |
| Output dir | `_site` | Built site (gitignored) |
| Template formats | njk, html, md | Processed file types |
| Passthrough | `assets`, `artist-template` | Copied as-is to output |

### Custom Filters

| Filter | Usage | Description |
|---|---|---|
| `year` | `{{ "" \| year }}` | Current 4-digit year |

### Shortcodes

| Shortcode | Usage | Description |
|---|---|---|
| `label` | `{% label "text" %}` | Renders a section label element |
| `feature` | `{% feature "icon", "text" %}` | Renders a feature list item |

### Collections

| Collection | Tag | Pages |
|---|---|---|
| `phase1` | `tags: phase1` | Active service pages |
| `phase2` | `tags: phase2` | Coming soon pages |
